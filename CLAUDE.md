# CLAUDE.md — تعليمات البناء لمشروع "وجهة أشوس"

هذا الملف موجّه مباشرة لك (Claude Code) وأنت تبني هذا المشروع. اقرأ هذا الملف كاملاً قبل أي كود. المرجعان الآخران بنفس المجلد إلزاميان أيضاً:
- `خطة_وجهة_أشوس_الشاملة.md` — كل الميزات بالتفصيل + مخطط قاعدة البيانات الكامل (SQL جاهز).
- `design.md` — كل توكنز التصميم (ألوان، طباعة، مسافات). ممنوع اختراع ألوان أو مقاسات خارج هذا الملف.

## عن المشروع
منصة عربية (RTL بالكامل) لطالب التوجيهي الفلسطيني (علمي/أدبي): جدولة أسبوعية + مهام يومية تلقائية + بومودورو + بنك أسئلة + خريطة تقدّم + نظام مكافأة بصري. الطالب هو صاحب القرار دائماً — لا شي يُفرض عليه تلقائياً بدون موافقة صريحة.

## الستاك (لا تغيّره بدون سبب قوي)
Next.js 14+ (App Router, TypeScript) · Tailwind CSS + shadcn/ui · Supabase (Postgres + Auth + Storage + RLS) · Dexie.js (IndexedDB, للعمل بدون إنترنت) · TanStack Query · Serwist (PWA/Service Worker) · Vercel (استضافة).

## قواعد غير قابلة للتفاوض
1. **RTL دائماً.** `<html dir="rtl" lang="ar">` من أول commit. لا تبني أي شاشة LTR "مؤقتاً".
2. **Dark mode هو الوضع الوحيد بالـMVP.** لا تبني وضع فاتح.
3. **كل توكن لوني/مسافة/خط لازم يجي من `design.md`.** ما في قيم hex أو px حرة بالكود.
4. **local-first إلزامي للميزات المحلية** (الجدولة بأنواعها الثلاثة، المهام، البومودورو، حالة المهمة، العد التنازلي، التكرار المتباعد) — أي تفاعل يُكتب بـ IndexedDB فوراً وتُحدَّث الواجهة قبل أي انتظار لشبكة، ثم يُزامَن بالخلفية. راجع قسم ب.5 بملف الخطة لتصنيف كل ميزة.
5. **RLS مفعّل على كل جدول فيه بيانات مستخدم** — لا استثناءات، حتى أثناء التطوير.
6. **النطاق كامل — لا شي "لاحق" إلا تطبيق الموبايل الحقيقي.** كل بند بقسم أ بملف الخطة (بما فيها الجدولة التلقائية، التنافسي، العد التنازلي، المحاكاة، التكرار المتباعد، صندوق الشكوك، وكل قوالب المكافأة الخمسة) مطلوب فعلياً — لا تفترض أي شي منها اختياري أو مؤجّل. **وضع أولياء الأمور ومجتمع الأسئلة أُزيلا نهائياً من النطاق — لا تبنِ أي أثر لهم.**
7. **أي جدول تلقائي (مساعد أو تلقائي بالكامل) لازم موافقة الطالب قبل التفعيل**، إلا لو فعّل بنفسه `auto_schedule_apply` من الإعدادات.
8. **كل نص واجهة بالعربية الفصحى المبسّطة** (لا عامية بالواجهة نفسها، العامية فقط كانت لغة النقاش بيني وبين المستخدم).

## ترتيب البناء المقترح (5 جلسات مركّزة — النطاق كبير فعلياً، لا تحاول ضغطه بجلسة أو جلستين على حساب الجودة)

### الجلسة 1 — الأساس الكامل
1. تهيئة المشروع (`create-next-app`, TypeScript, Tailwind, App Router) + shadcn/ui.
2. تطبيق كل توكنز `design.md` بـ `tailwind.config.ts` + `globals.css`.
3. ربط Supabase وتشغيل **كامل** SQL من قسم ب.3 بملف الخطة (كل الجداول، بما فيها التنافسي والمحاكاة وصندوق الشكوك — حتى لو واجهتها بجلسات لاحقة، المخطط يُبنى كاملاً من الأول لتفادي migrations متقطعة).
4. مصادقة كاملة (تسجيل/دخول/خروج/استعادة كلمة مرور) + `middleware.ts` + شاشة "اختر فرعك" الإجبارية.
5. بذر بيانات المواد لمادتين-ثلاث مواد تجريبية.
6. طبقة local-first (Dexie + طابور مزامنة) — الأساس اللي كل الجلسات الجاية تبني فوقه.

### الجلسة 2 — حلقة الاستخدام اليومي
1. الجدولة بأنظمتها الثلاثة (يدوي، مساعد، تلقائي بالكامل مع محرك الترجيح البسيط).
2. توليد "قائمة اليوم" + حالة المهمة الثلاثية.
3. بومودورو مربوط بكل مهمة.
4. **صندوق الشكوك** ملاصق لمؤقّت البومودورو (حقل نص سريع بدون مودال يقطع التركيز) + ملخّص أسبوعي بعدد الأسئلة غير المحلولة ضمن التذكيرات الدراسية (راجع أ.16).
5. وضع العد التنازلي (`exam_schedule` + منطق تحويل عرض المادة عند الاقتراب من الامتحان).
6. التكرار المتباعد (توليد مهام "مراجعة" تلقائياً عند إتمام درس).

### الجلسة 3 — المحتوى والإتقان
1. **سكربت استيراد بنك الأسئلة**: محتوى الأسئلة رح يجي بملفات جاهزة من صاحب المشروع (راجع أ.7) — أول خطوة هون افحص بنية الملفات الفعلية المرفوعة وابنِ الاستيراد على أساسها، لا تفترض صيغة قبل ما تشوفها. لو الملفات لسا ما وصلت، ابنِ الشاشات كاملة مع بيانات تجريبية (seed) بسيطة تكفي للاختبار، والاستيراد الحقيقي يشتغل بأي وقت لاحق بدون تعديل بالواجهة.
2. بنك الأسئلة كامل (عرض، تسجيل محاولات، شرح الحل، تحميل انتقائي للعمل بدون نت).
3. صفحة التقدّم الشاملة + الخريطة الحرارية (view محسوبة).
4. اختبارات المحاكاة الكاملة بزمن حقيقي (مؤقّت لا يتوقف، تسليم تلقائي، نتيجة مفصّلة).

### الجلسة 4 — التحفيز والمكافآت
1. **كل قوالب المكافأة الخمسة** فعلياً (خارطة فلسطين، النجوم والأبراج كافتراضي، بناء مدينة، شجرة/حديقة، بسيط) + شاشة تبديل القالب من الإعدادات بدون فقدان بيانات.
2. التذكيرات ثلاثية الطبقة + إشعارات متصفح (Web Push).

### الجلسة 5 — التنافسي + التلميع النهائي
1. لوحة الصدارة الاختيارية (بمؤشر التزام لا علامات) + التحديات الجماعية بكود دعوة.
2. PWA كامل (manifest، أيقونات، Service Worker Serwist).
3. تمرير شامل: كل شاشة على 375px أولاً، فحص RTL، فحص عمل كل ميزة "offline كامل" فعلياً بقطع الشبكة، تدقيق نهائي مقابل `design.md`.

**معيار نجاح كل جلسة:** لا تنتقل لجلسة تالية إلا وكل ميزة بالجلسة الحالية تشتغل فعلياً (مو مجرد واجهة بدون منطق خلفي) وتجتاز قائمة التحقق بالأسفل.

## أوامر أساسية
```bash
npm run dev        # تشغيل محلي
npm run build       # بناء الإنتاج — لازم ينجح بدون أخطاء قبل أي نشر
npm run lint         # ESLint
npx tsc --noEmit    # فحص TypeScript الصارم
```

## تحقّق قبل ما تعتبر أي ميزة "خلصت"
- تعمل RTL صح (لا نص منقلب أو محاذاة خاطئة).
- تعمل بدون إنترنت لو كانت من الميزات المحلية (اقطع الشبكة بأدوات المتصفح وجرّب).
- الألوان/الخطوط مطابقة `design.md` بدون قيم حرة.
- RLS مفعّل ومُختبر (جرّب تجيب بيانات مستخدم ثاني وتأكد إنها محجوبة).
- `npm run build` و `npx tsc --noEmit` ينجحوا بدون أخطاء.

لو ميزة مو موثّقة بوضوح بملف الخطة، أو قرار تصميم مو موجود بـ `design.md` — لا تخترع. علّق TODO واضح بالكود واستمر بباقي البناء، وأشِّر عليه بالخلاصة النهائية للجلسة.
