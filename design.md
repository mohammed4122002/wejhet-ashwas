# design.md — نظام التصميم | منصة "وجهة أشوس"

> هذا الملف مرجع تصميم إلزامي لأي جلسة Claude Code تبني بهذا المشروع. أي مكوّن جديد لازم يلتزم بالتوكنز (tokens) هون، مو ألوان أو مقاسات عشوائية.

## 1. الفكرة البصرية للعلامة

الشعار: هلال/قوس نحاسي محيط بكوكب متوهّج بتدرّج أحمر-برتقالي دافئ، وبداخله طريق يمتد نحو الأفق تحت الكوكب، وسهم/طائرة ورقية صغيرة عند الطرف العلوي، ونجمة إضاءة. الشكل الكلي أقرب لقلب/دلالة "وجهة" (pin location).

**الدلالة اللي نبني عليها التصميم:** رحلة الطالب نحو هدفه (القبول الجامعي/التوجيهي) — طريق واضح، وجهة مضيئة بالأفق، وتقدّم تدريجي (الكوكب/الطريق يُضاء أكتر كل ما اقترب). هاي نفس فلسفة "نظام المكافأة" اللي حددناها بخطة الميزات — الهوية البصرية والمنطق الوظيفي متوافقين من الأساس.

**الوضع الافتراضي للتطبيق: Dark Mode دائماً.** الشعار مصمم أصلاً على خلفية غامقة، والتدرّج الدافئ بيبرز فقط على خلفية غامقة. ما رح نبني وضع فاتح بالـMVP.

*(ملاحظة: الألوان تحتها مستخلصة بصرياً من الشعار المرفق، مو بقراءة بكسل دقيقة. أول ما يتوفر ملف الشعار الأصلي بصيغة عالية الجودة (SVG/PNG) بمجلد `public/brand/`، اعتمد أداة eyedropper عليه وعدّل القيم إذا لزم فرق طفيف.)*

## 2. لوحة الألوان (CSS Variables)

```css
:root {
  /* خلفيات */
  --bg-base: #150B0F;          /* الخلفية الأساسية للتطبيق كامل */
  --bg-surface: #221219;       /* بطاقات، أشرطة جانبية */
  --bg-surface-raised: #2E1820; /* عناصر مرفوعة (مودال، قائمة منسدلة) */
  --bg-overlay: rgba(21, 11, 15, 0.85);

  /* التدرّج الأساسي (هوية العلامة) */
  --brand-500: #E0603F;   /* اللون الأساسي — أزرار، عناوين، حالة "قيد الدراسة" */
  --brand-600: #C6472C;   /* هوفر/ضغط */
  --brand-400: #F08157;   /* أفتح، للتمييز الثانوي */
  --brand-glow: #FBC79A;  /* توهّج/highlight — الجزء العلوي من الكوكب بالشعار */

  /* النحاسي/الذهبي (القوس بالشعار) — نستخدمه كلون ثانوي/تمييز */
  --accent-copper: #D97757;
  --accent-gold: #E0A868;

  /* نصوص */
  --text-primary: #F5EDE8;     /* أبيض دافئ — النص الأساسي */
  --text-secondary: #B8A79E;   /* رمادي دافئ — نص ثانوي/وصفي */
  --text-muted: #7A6B64;       /* نص خافت جداً — تلميحات */
  --text-on-brand: #150B0F;    /* نص فوق خلفية brand-500 (غامق ليضمن تباين) */

  /* حالات دلالية (Kanban) */
  --status-todo: #7A6B64;       /* لسا */
  --status-in-progress: #E0603F; /* قيد الدراسة */
  --status-done: #4CAF7D;       /* تمّت — أخضر مريح، خارج عائلة العلامة بالكامل حتى يتميّز فوراً */

  /* حدود وفواصل */
  --border-subtle: rgba(245, 237, 232, 0.08);
  --border-strong: rgba(245, 237, 232, 0.16);
}
```

**قاعدة استخدام مهمة:** التدرّج البرتقالي-الأحمر (`--brand-*`) محجوز للعلامة والعناصر التفاعلية الأساسية (أزرار CTA، عناوين، حالة "قيد الدراسة"، تقدّم البومودورو). ما نستخدمه كخلفية كبيرة أو نص طويل — على خلفية غامقة كثير برتقالي بيتعب العين بجلسات مذاكرة طويلة. النص الطويل دائماً بـ`--text-primary` أو `--text-secondary`.

**تباين (Accessibility):** `--text-primary` (#F5EDE8) على `--bg-base` (#150B0F) نسبة تباين ~14:1 — ممتاز. `--brand-500` على `--bg-base` نسبة ~4.6:1 — كافي للنصوص الكبيرة والعناصر التفاعلية، لكن تجنّب استخدامه لنص صغير (<16px) على الخلفية الغامقة مباشرة؛ استخدم `--brand-400` بدلاً منه لو النص صغير.

## 3. الطباعة (Typography)

**الخط: [IBM Plex Sans Arabic](https://fonts.google.com/specimen/IBM+Plex+Sans+Arabic)** (عبر Google Fonts) — مجاني، أوزان كافية (400/500/600/700)، تصميم واضح ومريح لفترات قراءة طويلة، ودعم ممتاز للأرقام العربية والغربية معاً.

- استخدم **الأرقام الغربية (0-9)** بكل الواجهة (مؤقتات، نسب، تواريخ) — أوضح على الشاشة وأسرع قراءة بسياق دراسة/عد تنازلي.
- اتجاه الصفحة: `<html dir="rtl" lang="ar">` من الأساس — ما رح نبني نسخة LTR بالـMVP.

**السلّم (Scale):**

| الاستخدام | الحجم | الوزن |
|---|---|---|
| عنوان صفحة (H1) | 28px / 1.75rem | 700 |
| عنوان قسم (H2) | 22px / 1.375rem | 600 |
| عنوان بطاقة (H3) | 17px / 1.0625rem | 600 |
| نص أساسي | 15px / 0.9375rem | 400 |
| نص ثانوي/وصفي | 13px / 0.8125rem | 400 |
| أرقام كبيرة (مؤقت بومودورو، نسبة تقدّم) | 40px+ | 700 |

## 4. المسافات، الحواف، الظلال

- **وحدة المسافات الأساسية:** 4px (استخدم مضاعفاتها: 4/8/12/16/24/32/48).
- **الحواف الدائرية:** كروت وأزرار كبيرة `radius: 16px`، أزرار صغيرة/شارات `radius: 999px` (pill)، حقول إدخال `radius: 10px`. الشعار بشكله المستدير (قلب/هلال) يبرر ميل عام نحو حواف ناعمة بكل مكان — تجنّب الزوايا الحادة تماماً.
- **الظلال/التوهّج:** بدل ظل تقليدي أسود، استخدم "glow" بلون العلامة على العناصر النشطة/المُنجزة:
  ```css
  --glow-brand: 0 0 24px rgba(224, 96, 63, 0.35);
  --glow-success: 0 0 20px rgba(76, 175, 125, 0.3);
  ```
  يُستخدم هذا خصوصاً بلحظات المكافأة (فتح إنجاز جديد) وبعنصر البومودورو وهو شغّال — يربط بصرياً مع "الكوكب المتوهّج" بالشعار.

## 5. الحركة (Motion)

- انتقالات خفيفة وسريعة (150-250ms، ease-out) — التطبيق موجّه لأجهزة متوسطة/ضعيفة أحياناً (غزة/الضفة)، فتجنّب أي حركة ثقيلة أو مبنية على مكتبات رسوم كبيرة.
- لحظة "فتح مكافأة" فقط تستاهل حركة أوضح (نبضة توهّج + تكبير بسيط 1 → 1.05 → 1) لأنها لحظة تحفيزية مقصودة.
- لا حركة زخرفية دائمة (parallax، خلفيات متحركة) — تستهلك بطارية وبيانات بلا داعي.

## 6. المكوّنات الأساسية — إرشادات

### حالة المهمة (Kanban المبسط)
ثلاث حالات فقط، كل وحدة لها لون دلالي ثابت (قسم 2) — تمثيلها الأفضل **شارة صغيرة + لون خلفية خفيف** (10% opacity من لون الحالة) على بطاقة المهمة، مو ألوان صارخة كاملة.

### الخريطة الحرارية
شبكة (grid) من مربعات صغيرة بدرجات من `--bg-surface` (لسا ما بدأ) تدريجياً حتى `--brand-500` أو `--status-done` (حسب اخترنا نربطها بالتقدّم العام أو بالإتمام). تجنّب الأحمر التقليدي "خطر" — بما إن العلامة أصلاً حمراء-برتقالية، درجة التشبّع هي اللي بتفرّق (فاتح جداً = بداية، مشبّع بالكامل = إتقان).

### أنظمة المكافأة المتعددة — خمسة قوالب، كلها مبنية فعلياً بالـMVP
خارطة فلسطين / النجوم والأبراج / بناء مدينة / شجرة-حديقة / بسيط — كل قالب يستخدم نفس توكنز الألوان أعلاه بالضبط، الفرق بالرسم/الأيقونات فقط، مو بلوحة ألوان جديدة لكل قالب. هيك نضمن اتساق الهوية بغض النظر شو اختار الطالب.

**قالب "النجوم والأبراج" هو الافتراضي المقترح عند أول تسجيل** — لأنه الأقرب بصرياً لعناصر الشعار نفسه (الكوكب المتوهّج، النجمة، القوس). خلفية سماء ليلية غامقة (`--bg-base`)، النجوم بلون `--brand-glow` (#FBC79A) بتوهّج خفيف (`--glow-brand`)، والخطوط الواصلة بين نجوم الوحدة المكتملة بلون `--accent-copper`.

**قالب "خارطة فلسطين"** يستخدم نفس الخلفية الغامقة، وكل مدينة "معتمة" بلون `--bg-surface`، وتتدرّج نحو `--brand-500` مع التوهّج (`--glow-brand`) كل ما زاد التقدّم العام — لا نضيف أي لون أخضر/أحمر علم إضافي حتى لا نكسر اتساق اللوحة، الدلالة الوطنية تجي من شكل الخارطة نفسها لا من ألوان جديدة.

### صندوق الشكوك (Doubt Box) — أثناء البومودورو
عنصر ثابت وصغير جداً ملاصق لمؤقّت البومودورو (شارة/زر دائري بأيقونة "؟" بلون `--accent-copper`)، بالضغط عليه يفرد حقل نص واحد فقط مكانه مباشرة (inline، بدون مودال أو تنقّل صفحة) مع زر حفظ صغير. الهدف تصميمياً: أقل احتكاك ممكن — أي خطوة إضافية (مودال، تحميل صفحة، حقول متعددة) تتعارض مع الغرض من الميزة نفسها (الحفاظ على تركيز جلسة المذاكرة).

### الأزرار
- **أساسي (Primary):** خلفية `--brand-500`، نص `--text-on-brand`، حواف 10px، هوفر → `--brand-600`.
- **ثانوي:** حدود `--border-strong`، خلفية شفافة، نص `--text-primary`.
- **نصّي (Ghost):** بدون خلفية ولا حدود، نص `--brand-400`.

### الأيقونات
مكتبة [lucide-react](https://lucide.dev) — خطوط بسيطة (outline)، متوافقة مع الطابع الهادئ للشعار، خفيفة على الأداء.

## 7. الشعار — أصول مطلوبة قبل البناء

الشعار المرفق بالمحادثة صورة مرجعية فقط (وصفه بصرياً موثّق بقسم 1). **قبل بدء جلسة Claude Code، لازم تجهيز:**

- نسخة SVG أو PNG شفافة عالية الدقة من الشعار → توضع بـ `public/brand/logo.svg` (أو png).
- نسخة مبسّطة/أيقونة فقط (بدون النص) لاستخدامها كـ favicon و PWA icon (192×192، 512×512).
- تأكيد نهائي على النص العربي بالشعار (كان غير واضح بالنسخة المولّدة بالذكاء الاصطناعي) — إذا حبيت تجدّد تصميم الشعار بخط عربي سليم 100%، هاي مهمة منفصلة نقدر نعملها قبل التسليم لـ Claude Code.
